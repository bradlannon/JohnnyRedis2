## Integration Gaps

### 1. CommandPayload.value Schema Mismatch (SCHED-01, SCHED-02)

**From:** `packages/web/src/components/SchedulerUI.tsx` (local `CommandPayload` with `value?` optional)
**To:** `packages/shared/src/payloads.ts` (`CommandPayload` with `value` required)

SchedulerUI can submit a schedule without a `value` field (e.g., "turn LED on"). The server validates against the shared `CommandPayload` Zod schema which requires `value`, returning 400.

**Fix (choose one):**
- Make `value` optional in shared `CommandPayload`: `value: z.union([...]).optional()`
- Or require value in SchedulerUI form (disable Save when empty for non-toggle actions)

### 2. CORS Preflight Missing (Production Risk)

**From:** `packages/server/src/index.ts` (manual CORS headers, no OPTIONS handler)
**To:** Browser CORS preflight for POST/PUT with `application/json`

In production (Hostinger), `POST /command` and `/api/schedules` endpoints will fail CORS preflight because there is no `OPTIONS` handler. The Vite dev proxy masks this entirely.

**Fix:** Add `app.options('*', cors())` or replace manual headers with the `cors` npm middleware.

### 3. TypeScript Compile Error (Phase 3)

**From:** `packages/server/src/index.ts` line 5 (`import.meta.url`)
**To:** `packages/server/package.json` (missing `"type": "module"`)

`tsc --noEmit` fails with TS1470. Runtime is unaffected (tsx handles ESM transparently).

**Fix:** Add `"type": "module"` to `packages/server/package.json`.
