## Integration Gaps

### 1. CommandPayload.value Schema Mismatch (SCHED-01, SCHED-02)

**From:** `packages/web/src/components/SchedulerUI.tsx:119`
**To:** `packages/shared/src/payloads.ts:19` (`CommandPayload` Zod schema)

SchedulerUI conditionally omits `value` when form field is empty: `...(form.value !== '' ? { value: ... } : {})`. The shared CommandPayload requires `value: z.union([z.string(), z.number(), z.boolean()])` — no `.optional()`. Any schedule saved without a value returns HTTP 400.

**Fix:** Add `.optional()` to value in `CommandPayload`:
```ts
value: z.union([z.string(), z.number(), z.boolean()]).optional()
```

### 2. CORS Preflight Missing (Production Risk)

**From:** `packages/server/src/index.ts` (manual CORS headers, no OPTIONS handler)
**To:** Browser CORS preflight for POST/PUT with `application/json`

In production (Hostinger), `POST /command` and `/api/schedules` will fail CORS preflight because there is no `OPTIONS` handler. The Vite dev proxy masks this.

**Fix:** Add `cors` middleware or explicit OPTIONS handler.
