       - Uses `useRef<HTMLVideoElement>` and `useEffect` for hls.js lifecycle:
         - If src is null, set status='offline'
         - If `Hls.isSupported()`: create Hls instance with `liveSyncDurationCount: 3`, loadSource, attachMedia, listen for MANIFEST_PARSED (set playing, autoplay), ERROR with fatal (set error/offline)
         - Else if Safari native HLS: set src directly on video element
         - Cleanup: destroy hls instance on unmount/src change
       - Retry logic: if status is 'error' or 'offline' and src is not null, retry every 15 seconds via setTimeout
       - Renders:
         - When playing: `<video ref={videoRef} className="w-full aspect-video bg-black rounded-lg" muted playsInline />` with "LIVE . ~5s delay" badge overlaid (absolute positioned, top-right, green dot + text)
         - When offline/error/connecting: dark box (`bg-gray-900`) with camera icon (SVG), status text ("Camera offline" / "Connecting..." / "Stream unavailable"), matching StatusBadge visual pattern
       - Expose `videoRef` for fullscreen and snapshot controls from parent

    3. Create `packages/web/src/components/CameraSection.tsx`:
       - Props: none (reads camera URLs from env vars)
       - Read URLs: `const webcamUrl = import.meta.env.VITE_CAMERA_WEBCAM_URL ?? null` and `const piCameraUrl = import.meta.env.VITE_CAMERA_PI_URL ?? null`
       - Internal state: `activeTab: 'webcam' | 'pi_camera'`, default 'webcam'
       - Renders:
         - Section header "Camera" with tab buttons ("Webcam" / "Pi Camera") — styled same as ChartSection window selector
         - Only the active tab's CameraFeed renders (lazy — saves bandwidth per user decision)
         - Fullscreen button: calls `videoRef.current?.requestFullscreen()` — needs ref forwarded from CameraFeed
         - Snapshot button: draws video frame to offscreen canvas, triggers download as PNG
         - If BOTH urls are null, show a single placeholder: "No camera URLs configured"

    4. Create `packages/web/src/components/CameraFeed.test.tsx`:

    5. Create `packages/web/src/hooks/useNotifications.ts`:
       - `useNotifications(motionValue: number | null, enabled: boolean)` hook
       - Uses `useRef` to track previous motion value
       - `requestPermission()` function — calls `Notification.requestPermission()`, must be invoked from user gesture (click handler)
       - `useEffect` watching motionValue + enabled:
         - If !enabled or permission !== 'granted', skip
         - If motionValue === 1 AND prevMotion.current !== 1 (rising edge only), fire `new Notification('JohnnyRedis', { body: 'Motion detected!', icon: '/favicon.ico' })`
         - 30-second cooldown: track `lastNotifiedAt` ref, skip if within cooldown
         - Update prevMotion.current
       - Return `{ requestPermission, permission: Notification.permission, supported: 'Notification' in window }`

       - Test: requestPermission calls Notification.requestPermission
