    2. Create `packages/web/src/lib/chartSetup.ts`:
       - Register all required Chart.js components: CategoryScale, LinearScale, TimeSeriesScale, PointElement, LineElement, Title, Tooltip, Legend, Filler
       - Register chartjs-adapter-date-fns (import side-effect: `import 'chartjs-adapter-date-fns'`)
       - Register CrosshairPlugin from chartjs-plugin-crosshair
       - Register ZoomPlugin from chartjs-plugin-zoom

       - NOTE: If `chartjs-plugin-crosshair` does not export `CrosshairPlugin` as a named export, try the default export. Check the actual package exports. The research warns this plugin is sparsely maintained — if the import fails, fall back to: `import CrosshairPlugin from 'chartjs-plugin-crosshair'`

    3. Create `packages/web/src/hooks/useHistory.ts`:
       - `useHistory(device: string, board: string, window: '1h' | '24h' | '7d')` hook
       - On mount and when device/board/window changes, fetch from `/api/sensors/${device}/history?board=${board}&window=${window}`
       - Return `{ data: Array<{x: number, y: number}>, aggregated: boolean, loading: boolean, error: string | null }`
       - Transform response: for raw data, map `{createdAt, value}` to `{x: new Date(createdAt).getTime(), y: value}`; for aggregated, map `{bucket, avgValue}` to `{x: new Date(bucket).getTime(), y: avgValue}`
       - Expose `appendPoint(ts: number, value: number)` function for real-time SSE updates — pushes to data array via setState, trims to max 500 points
       - Track `userInteracted` boolean (set true on zoom/pan, false on window change) — when true, do NOT auto-scroll the chart's x-axis

    4. Create `packages/web/src/components/SensorChart.tsx`:
       - Props: `{ device: string, board: string, label: string, color: string, stepped?: boolean, window: '1h' | '24h' | '7d', latestValue?: { ts: number, value: number } | null }`
       - Uses `useHistory(device, board, window)` internally
       - Renders `<Line ref={chartRef} data={chartData} options={chartOptions} />` from react-chartjs-2
       - Chart options per user decisions:
         - `stepped: boolean` prop controls `stepped: 'before'` on the dataset (for motion/button) vs `tension: 0.3` (for photoresistor/potentiometer)
         - Time series x-axis with appropriate time unit per window (1h: 'minute', 24h: 'hour', 7d: 'day')
         - Zoom plugin: `zoom: { wheel: { enabled: true }, pinch: { enabled: true }, mode: 'x' }`, `pan: { enabled: true, mode: 'x' }`
         - Crosshair plugin: `sync: { enabled: true, group: 1, suppressTooltips: false }`, `line: { color: '#6b7280', width: 1 }`, `zoom: { enabled: false }` (zoom handled by zoom plugin)
         - Tooltip: show "avg" label when `aggregated` is true (callback: `label: (ctx) => aggregated ? \`avg: ${ctx.parsed.y.toFixed(1)}\` : \`${ctx.parsed.y}\``)
       - Real-time append: useEffect watching `latestValue` — when it changes and `window === '1h'` (raw data), call `appendPoint()`. For 24h/7d, skip (aggregated data would be stale).
       - On zoom/pan interaction, set userInteracted=true via zoom plugin's `onZoom` and `onPan` callbacks
       - When userInteracted is false and new data appends, call `chart.options.scales.x.max = Date.now()` to auto-scroll

    5. Create `packages/web/src/components/SensorChart.test.tsx`:
       - Mock useHistory to return controlled data
       - Test: renders without crashing with empty data
       - Test: renders with data points (check canvas element exists)
       - Test: stepped prop applies to binary sensors
