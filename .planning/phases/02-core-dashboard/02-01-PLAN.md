<objective>
Hub serial-to-MQTT bridge: serialport v13 reads Arduino JSON over stable udev symlinks, reconnects on USB unplug, publishes sensor data to MQTT, subscribes to command topics and writes commands to Arduino serial.

Purpose: This is the physical-world interface -- the hub translates between Arduino hardware (serial JSON lines) and the MQTT message bus that the server consumes. Without this, no sensor data flows and no commands reach hardware.

Output: Hub process that reads Arduino sensors over serial, publishes to MQTT, receives commands from MQTT, and writes them to Arduino serial -- with reconnect resilience.
</objective>
