<objective>
Server MQTT-to-SSE bridge with state cache and command API: MQTT subscriber receives hub sensor/status data, caches last values, broadcasts to SSE clients, replays full state on new client connect, and exposes POST /command for browser-to-hardware control.

Purpose: This is the server-side bridge that makes real-time dashboard possible. It connects the hub's MQTT messages to browser SSE clients, maintains state for instant page loads (INFRA-04), syncs across tabs (INFRA-05), and provides the command pathway from browser to hardware.

Output: Server with MQTT subscriber, last-value state cache, SSE state replay, and POST /command endpoint.
